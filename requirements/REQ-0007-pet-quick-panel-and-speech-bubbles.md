# REQ-0007：宠物点击、气泡与快捷功能面板

## 基本信息

- 状态：待验收
- 创建日期：2026-07-23
- 最后更新：2026-07-23
- 负责人：待定

## 背景

当前宠物窗口已经支持拖动、鼠标悬停、看向鼠标、随机动作和托盘设置，但用户无法直接点击宠物执行操作。后续规划中的日常周常、开服状态、定时提醒和宠物互动也缺少统一、直观的桌面入口。

如果继续把功能只放在托盘或完整设置窗口中，宠物本身仍然只是动画展示；如果直接扩大现有宠物窗口来容纳面板和气泡，扩大的透明区域会拦截桌面鼠标操作，并可能影响拖动、观察范围、窗口位置保存和缩放。

因此需要建立一套围绕宠物的轻量交互入口：用户单击宠物时打开快捷功能面板，应用可在宠物附近显示短时气泡，并为后续实用模块提供稳定的入口和消息展示能力。

## 目标

- 在不破坏现有拖动手感的前提下，可靠区分单击宠物与拖动宠物。
- 用户单击宠物后，可以在宠物附近打开一个轻量快捷功能面板。
- 应用可以用不抢焦点、可自动消失的气泡提供简短反馈。
- 面板和气泡在多显示器、不同缩放比例及屏幕边缘位置下均完整可见。
- 建立可复用的面板入口和气泡消息边界，供后续日常周常、开服状态及提醒需求接入。
- 保持宠物窗口、辅助窗口、动画行为和平台能力之间的清晰边界。

## 用户价值

- 用户可以直接与宠物交互，不必先寻找系统托盘。
- 常用功能可以在桌面上快速打开，并保持与宠物的视觉关联。
- 后续提醒和任务完成反馈可以通过宠物表达，而不只是普通系统通知。
- 功能入口统一后，新增模块不需要重复设计新的窗口唤起方式。

## 范围

### 包含

- 单击、拖动手势的区分与冲突处理。
- 单击宠物打开或关闭快捷功能面板。
- 面板的辅助窗口、定位、显示、隐藏和焦点管理。
- 气泡的展示、排队、去重、超时和定位。
- 当前版本可实际使用的宠物互动与设置入口。
- 面板、气泡与拖动、悬停、观察、随机动作之间的优先级。
- 气泡开关及相关本地配置迁移。
- Windows 10/11 和 macOS 开发环境下的多屏、DPI 与屏幕边缘处理。
- 核心手势、窗口定位和气泡队列的自动化测试。

### 不包含

- 日常周常清单及完成状态管理。
- JX3BOX 或其他第三方联网数据接入。
- 开服状态、官方公告或活动日历。
- 定时提醒、系统通知和勿扰时段。
- 多角色、账号、云同步或跨设备同步。
- AI 对话、自由聊天或语音交互。
- 亲密度、饱食度、货币、商店和复杂养成系统。
- 新增或修改宠物美术资源。
- 全局键盘监听、全局鼠标事件钩子或游戏进程读取。
- 插件系统或允许第三方代码动态注册面板入口。
- 为尚未实现的功能展示不可点击的“敬请期待”占位入口。

## 名词定义

- **宠物窗口**：当前显示并渲染黄鸡的透明无边框主窗口。
- **辅助窗口**：用于承载气泡或快捷功能面板的独立轻量窗口。
- **气泡模式**：辅助窗口仅展示一条短时文本消息，不主动抢占当前应用焦点。
- **面板模式**：辅助窗口展示可操作的快捷功能内容，并允许键盘焦点进入。
- **直接互动动作**：用户在面板中主动触发的 `waving`、`review` 等现有动作。
- **主动气泡**：非用户当次点击直接触发、由应用主动展示的提示。

## 核心用户场景

### 场景一：打开快捷面板

1. 用户将鼠标移动到宠物上。
2. 用户按下并在未超过拖动阈值的情况下松开鼠标左键。
3. 宠物播放一次 `waving` 动作。
4. 快捷功能面板出现在宠物附近且完整位于当前显示器工作区内。
5. 用户可以触发宠物动作、打开设置或关闭面板。

### 场景二：拖动宠物

1. 用户按下宠物并移动鼠标。
2. 位移达到拖动阈值后进入系统原生拖窗。
3. 已显示的气泡或面板立即关闭。
4. 宠物继续按照已有规则播放左右跑动动画。
5. 松开鼠标后恢复已有悬停、观察、随机动作和待机行为，不打开面板。

### 场景三：显示气泡

1. 用户在面板中点击“打个招呼”。
2. 宠物播放 `waving`，辅助窗口切换为气泡模式。
3. 气泡显示一句本地预置的短文本。
4. 气泡在约定时间后自动消失，不改变用户当前使用的其他应用焦点。

### 场景四：宠物靠近屏幕边缘

1. 用户将宠物拖到显示器左侧、右侧、顶部或底部。
2. 用户单击宠物或触发气泡。
3. 辅助窗口自动选择有空间的一侧，并限制在当前显示器工作区安全边距内。
4. 面板和气泡均不得跨出可用工作区或被任务栏遮挡。

## 需求说明

### 1. 点击与拖动判定

- 当前 `pointerdown` 后立即启动原生拖窗的行为需要调整为“达到拖动阈值后再开始拖动”。
- 鼠标左键按下时记录起始指针位置，但不立即打开面板。
- 指针相对起始位置的欧氏距离未达到 `4` 个逻辑像素，并在宠物区域内松开时，判定为单击。
- 指针位移达到或超过 `4` 个逻辑像素时，判定为拖动，并调用系统原生拖窗。
- 一次交互一旦判定为拖动，即使最终位置接近起点，也不得再触发单击。
- 拖动结束后的同一次 `pointerup` 不得打开面板。
- `pointercancel`、窗口失焦或拖动调用失败时必须清理本次手势状态。
- 不使用双击、长按或右键作为本需求的必要操作。
- 浏览器开发模式无法执行系统原生拖窗时，仍应能够验证单击判定和面板前端行为。

### 2. 辅助窗口

- 气泡和快捷面板使用宠物窗口之外的辅助窗口承载，不能通过永久扩大宠物主窗口实现。
- 首选复用同一个辅助窗口，在“隐藏、气泡、面板”三种模式间切换，避免同时创建多个 WebView。
- 辅助窗口必须无系统边框、不可最大化、不可调整大小、不显示在任务栏。
- 辅助窗口背景允许透明，但可点击区域必须与可见内容一致，尽量减少透明区域对桌面鼠标的遮挡。
- 辅助窗口默认隐藏；应用启动时可以预创建以降低首次打开延迟，但隐藏时不得执行动画计时、轮询或其他高频任务。
- 宠物被托盘隐藏或应用退出时，辅助窗口必须同步隐藏或销毁。
- 打开完整设置窗口时，快捷面板关闭。
- 宠物缩放、移动、恢复默认位置或跨显示器后，再次打开辅助窗口时必须使用最新窗口位置。
- 辅助窗口创建或显示失败时，不得影响宠物拖动、托盘和完整设置窗口的正常使用。

### 3. 快捷功能面板

- 单击宠物时切换面板显示状态：隐藏时打开，已打开时关闭。
- 面板模式推荐尺寸为 `320 × 300` 逻辑像素；实现可根据最终内容在不超过 `360 × 420` 的范围内调整。
- 面板至少包含：
  - 当前宠物名称与简短状态；
  - “打个招呼”：播放 `waving` 并显示本地气泡；
  - “随机动作”：从当前资源包已有且适合互动的动作中播放一次；
  - “窗口置顶”当前状态及快捷开关；
  - “宠物设置”：打开现有完整设置窗口；
  - 明确的关闭按钮。
- 面板不得展示尚未实现、不可操作的日常、开服、提醒等占位卡片。
- 后续需求可以在面板中增加入口，但每个入口仍需对应独立需求文档。
- 用户点击面板外部导致辅助窗口失去焦点时，面板自动关闭。
- 用户按 `Escape` 时关闭面板。
- 再次单击宠物、开始拖动宠物或从托盘隐藏宠物时关闭面板。
- 面板打开后可以接收键盘焦点；`Tab` 顺序必须与视觉顺序一致，按钮支持 `Enter` 和 `Space`。
- 面板不得保存或恢复上次打开状态，应用启动时始终保持关闭。

### 4. 气泡系统

- 气泡仅接受纯文本，不解析 HTML，不加载远程图片。
- 单条文本建议不超过 `60` 个中文字符，最多显示 `3` 行；超出部分使用省略号并写入开发日志。
- 同一时间只显示一个气泡。
- 默认展示时间为 `4` 秒；允许调用方在 `2`～`8` 秒范围内指定时长。
- 最多等待 `3` 条气泡；队列已满时丢弃最早的普通消息，不能无限累积。
- 相同消息在 `30` 秒内重复触发时只保留一次。
- 面板优先级高于气泡：打开面板时清空当前普通气泡并切换为面板模式。
- 开始拖动宠物、隐藏宠物或应用退出时立即关闭气泡。
- 气泡显示不得主动抢占用户正在使用的其他应用焦点。
- 用户点击气泡时可以提前关闭气泡；首版不要求气泡包含多个操作按钮。
- 气泡关闭后，辅助窗口不得继续占用桌面点击区域。
- 本需求不主动实现定时提醒；首版气泡只用于用户直接互动、操作成功和可恢复错误的简短反馈。

### 5. 气泡设置与持久化

- 设置窗口新增“显示宠物气泡”开关，默认开启。
- 关闭后不展示主动气泡和互动台词；相关操作仍执行，并在面板内提供必要的状态反馈。
- 配置结构需要向后兼容现有版本，已有窗口位置、缩放、置顶状态和当前宠物不得丢失。
- 若升级配置版本，迁移只能执行一次，并提供 Rust 单元测试。
- 气泡内容、展示时间和用户交互记录不写入配置，不建立聊天历史。

### 6. 窗口定位规则

- 定位以宠物窗口在当前显示器工作区内的物理矩形为基础，再根据当前显示器缩放比例换算。
- 面板与宠物窗口之间保留约 `12` 个逻辑像素间距。
- 面板优先显示在宠物左侧；左侧空间不足时显示在右侧。
- 左右均不足时，选择可见面积更大的一侧，并将面板限制在当前显示器工作区内。
- 面板垂直方向默认与宠物底部对齐，再限制在工作区上下边界 `12` 个逻辑像素安全边距内。
- 气泡优先显示在宠物上方并水平居中；上方空间不足时显示在下方。
- 定位必须使用工作区而不是完整屏幕尺寸，避免被 Windows 任务栏遮挡。
- 宠物跨显示器时使用宠物中心点所在的显示器；无法识别时退回主显示器。
- 辅助窗口不得因为坐标换算错误在不同 DPI 显示器之间闪动或漂移。
- 定位算法应尽量实现为不依赖 Vue 或 Tauri 的纯函数，并覆盖单元测试。

### 7. 动画与行为优先级

新增直接互动后，顶层优先级调整为：

```text
Dragging（左右跑动）
  > DirectInteraction（面板触发动作）
  > Hovering（悬停 jumping）
  > CursorLooking（16方向观察）
  > Acting（随机动作）
  > Idle（待机）
```

- 单击打开面板时播放一次 `waving`，播放完成后根据鼠标位置恢复悬停、观察或待机。
- 面板可见本身不持续占用动画状态，也不阻止后续普通随机动作。
- 面板触发直接互动动作时暂停随机动作调度。
- 直接互动动作播放结束后，从 `idle` 重新调度，不继续被打断的随机动作剩余帧。
- 开始拖动时立即打断直接互动动作，并关闭面板或气泡。
- 气泡本身不强制触发动画；是否联动动作由触发气泡的交互决定。

### 8. 窗口层级与焦点

- 宠物设置为置顶时，辅助窗口打开期间应保持在普通窗口之上并与宠物保持一致的使用体验。
- 宠物关闭置顶时，辅助窗口不得通过持续轮询强制抢占系统最前层。
- 气泡显示时不主动聚焦辅助窗口。
- 面板由用户单击宠物打开，可以获得焦点以支持键盘操作。
- 面板关闭后不强制把焦点切回宠物窗口或其他应用。
- 不注册全局快捷键，不监听全局键盘输入。

### 9. 视觉与文案

- 面板视觉延续现有设置窗口的暖黄色、圆角和轻量阴影风格。
- 快捷面板的宠物头像和设置窗口的当前宠物图标必须展示黄鸡资源本身，不使用系统 Emoji 代替，且两个入口复用同一头像组件。
- 气泡应清晰指向宠物，但不得依赖新的位图资源；箭头可由 CSS 绘制。
- 面板在 100% Windows 系统缩放下正文不得小于 `12px`，主要操作文字不得小于 `14px`。
- 明暗对比需要保证常用文本和按钮清晰可读。
- 面板不使用大面积持续动画；在系统开启“减少动态效果”时关闭非必要过渡。
- 首版互动台词使用少量本地固定文本，并保持简短、友好，不冒充官方游戏提示。
- 随机动作从一个动作切回 `idle` 时必须保持正面朝向；同一动作的连续帧不得在无转身过渡的情况下从正面或侧面瞬间切换到背面。
- 图集中包含独立姿势而非连续动作的行，应允许通过显式帧顺序选择、重复或跳过源帧；首版将 `failed` 调整为正面站立、坐下、趴下后按相反顺序恢复，不播放背面姿势。

### 10. 错误处理与降级

- 辅助窗口不可用时，宠物窗口继续运行，托盘和设置入口保持可用。
- 打开设置失败、切换置顶失败或播放动作失败时，在面板内展示简短错误，不使面板崩溃。
- 窗口定位信息不可用时，将辅助窗口放置在主显示器工作区中央附近，并保证可见。
- 所有计时器、窗口事件监听和异步任务必须在组件卸载、窗口隐藏或应用退出时正确清理。
- 本需求不增加遥测或远程错误上报。

## 数据与接口边界

### 本地配置新增字段

字段名称可在实现时按 Rust 命名规范调整，但语义固定：

```text
speech_bubbles_enabled: boolean
```

- 默认值：`true`。
- 只保存开关，不保存气泡内容或历史。

### 建议的前端内部接口

具体名称可在实现时调整，但至少需要清晰区分：

```text
toggleQuickPanel()
closeQuickPanel()
showSpeechBubble(message, durationMs?)
hideSpeechBubble()
playDirectInteraction(animationName)
```

- 气泡调用只接受受控纯文本。
- 窗口创建、定位、显示和隐藏集中在平台适配层，不允许业务组件散落调用任意 Tauri 窗口 API。

## 性能要求

- 辅助窗口已经预创建时，单击后 `200` 毫秒内应开始显示面板。
- 未预创建时，首次打开应在 `500` 毫秒内给出可见反馈。
- 用户触发气泡后 `200` 毫秒内应开始显示。
- 辅助窗口隐藏时不得保留气泡倒计时、定位轮询或动画循环。
- 连续打开关闭面板 `100` 次后，WebView 数量不得增长，事件监听器和计时器不得持续累积。
- 持续运行 `30` 分钟并进行拖动、面板和气泡交互后，无明显 CPU 异常占用或内存持续增长。

## 隐私与安全

- 不新增联网请求。
- 不采集、记录或上传点击位置、鼠标轨迹和面板使用记录。
- 不使用全局鼠标钩子；只处理宠物窗口和辅助窗口内的指针事件，以及现有拖动期间的受控状态查询。
- 气泡内容不接受 HTML，避免脚本或样式注入。
- Tauri 命令只暴露创建、定位、显示和隐藏辅助窗口所需的最小能力。
- 辅助窗口使用独立最小权限配置，不复用不必要的设置窗口或宠物窗口权限。

## 验收标准

### 点击与拖动

- [ ] 原地单击宠物能够打开面板，不会移动宠物窗口。
- [ ] 指针位移达到拖动阈值后只执行拖动，不会在松开时误开面板。
- [ ] 连续快速单击和拖动各 `20` 次，无明显误判。
- [ ] 原生拖动的左右跑动动画及松开恢复行为与现有版本一致。
- [ ] `pointercancel`、窗口失焦和拖动失败后可以正常进行下一次点击或拖动。

### 快捷面板

- [ ] 面板包含宠物信息、打招呼、随机动作、置顶开关、设置和关闭入口，所有入口均可实际使用。
- [x] 快捷面板和设置窗口均展示由当前黄鸡图集裁切出的头像，不再出现系统小鸡 Emoji。
- [ ] 再次单击宠物、点击关闭、按 `Escape`、面板失焦、开始拖动或隐藏宠物时，面板正确关闭。
- [ ] 面板支持完整的键盘 `Tab`、`Enter`、`Space` 和 `Escape` 操作。

## 实施建议

本节用于指导讨论，不代表需求确认前可以开始修改主体代码。

- 将点击/拖动手势判定实现为纯 TypeScript 状态机，避免继续在 Vue 组件中累积布尔状态。
- 将辅助窗口位置计算实现为纯函数，输入宠物矩形、辅助窗口尺寸、显示器工作区和缩放比例，输出受限位置。
- 将气泡队列实现为独立核心模块，负责去重、容量和生命周期，不直接依赖 Vue 或 Tauri。
- 使用平台适配层统一封装辅助窗口创建、模式切换、定位、显示、隐藏和层级处理。
- 面板与设置窗口复用现有偏好读取和写入入口，不直接访问配置文件。
- 在 Windows 实机重点复测透明辅助窗口的阴影、焦点、任务栏和不同 DPI 定位。
- 复用黄鸡图集的 `idle` 第一帧实现轻量头像组件，不新增重复位图或 Emoji 回退图标。
- 为图集动画增加可选显式帧列顺序；`failed` 使用 `0 → 1 → 2 → 1 → 0`，避免把背面等独立姿势串成连续动作。
